# Spec Kit (GitHub) — заметки для Craftsman

Источник: “Spec Kit” (GitHub) — набор шаблонов/процесса для Spec‑Driven Development + CLI `specify`.

## Коротко: что это
- Это **не** редактор и не UI‑библиотека.
- Это процесс: **конституция проекта → спека → план → задачи → реализация → чеклисты/анализ**.
- Лицензия: MIT (в оригинальном репозитории).

## Что мы берём (адаптация под Craftsman)
Мы используем идеи, но держим процесс лёгким (инди/MVP):

1) **Принципы/ограничения как “конституция”**
- У нас это: `agents.md` + `docs/CONTEXT.md`.

2) **Единый формат ТЗ**
- У нас это: `docs/templates/FEATURE_TZ.md`.

3) **“Max 3 уточняющих вопроса”**
- Если ТЗ недоописано — задаём максимум 3 high‑impact вопроса, иначе начинаются бесконечные уточнения.

4) **Регресс‑чеклист как часть задачи**
- У нас это: `docs/templates/REGRESSION_CHECKLIST.md`.

5) **Крупные изменения — этапами**
- Особенно “ядро”: редактор/рендер/undo/сохранение.

## Что мы НЕ берём
- CLI `specify` как зависимость/инструмент проекта (лишнее для продукта).
- Сетевые шаги/скачивание шаблонов на стороне приложения.
- “Корпоративный” оверхед (большие артефакты ради артефактов).

## Как применять на практике
1) Пишем ТЗ по шаблону.
2) Делаем максимум 3 уточнения (если нужно).
3) Дробим на этапы и проверяем регрессии после каждого этапа.
4) Итог фиксируем в `docs/WORKLOG.md`, а устойчивые правила — в `docs/CONTEXT.md`.
